# GymLog

> Android-first Flutter app for fast gym set logging (3-tap workflow). Offline-only. Vietnamese + English.

## Tech Stack

| Layer | Technology | Version |
|-------|-----------|---------|
| Framework | Flutter (Dart) | SDK ^3.11.5 |
| State | flutter_riverpod | ^2.5.1 |
| Database | Drift (SQLite) | ^2.22.0 |
| Routing | GoRouter | ^14.0.0 |
| Charts | fl_chart | ^0.68.0 |
| Export | excel | ^4.0.6 |
| Fonts | google_fonts (Bebas Neue, Inter, JetBrains Mono) | ^6.2.1 |

## Dev Commands

```bash
# Install dependencies
flutter pub get

# Run code generation (Drift database.g.dart)
flutter pub run build_runner build --delete-conflicting-outputs

# Run on emulator/device
flutter run

# Analyze
flutter analyze

# Build APK
flutter build apk
```

## Core Logic Summary

- **1RM Estimation:** Epley formula: `weight * (1 + reps / 30)`
- **Smart Rest Timer:** Base (120s compound / 60s isolation) + intensity bonus (+60s if >85%, +30s if >70%) + set fatigue (`(set-1)*10s`), clamped 30-300s
- **PR Detection:** On each set log, compare new est. 1RM against `PersonalRecords.bestEstimated1rm`. Upsert if higher.
- **Export:** XLSX via `excel` package, saved to Android Downloads via native MediaStore MethodChannel (`com.gymsetlogger/storage`)

## Key Constraints

- **NEVER** modify `database.g.dart` manually — it's generated by `build_runner`.
- **NEVER** change table schemas without bumping `schemaVersion` and adding a migration.
- **Database file path:** `appDocuments/gymlog/gymlog.sqlite` — do not hardcode alternative paths.
- **Colors:** Use `AppColors` constants from `lib/app/theme.dart`. Do not hardcode hex values in widgets.
- **Exercise muscle groups:** Must be one of: `chest`, `back`, `shoulders`, `arms`, `legs`, `core`.
- **Exercise types:** Must be one of: `compound`, `isolation`, `bodyweight`.
- **Rest timer range:** 30-300 seconds — never allow values outside this range.
- **Weight picker:** 0.0-300.0 kg, 0.5 kg increments. Reps: 1-50.
- **No ViewModel layer exists.** Business logic lives in widget state. Do not introduce a ViewModel pattern without team approval.
